Most users self-host; we develop against Cloud where every secret/key/env var is already provisioned. Before merging any feature that reads a new env var, secret, key, piece auth method, or DB extension/schema dependency, default to **zero setup** (auto-provision > derive > visibly-gate > manual setup as last resort). Never ship a feature that looks enabled in the UI but is silently broken without manual setup. Read `docs/handbook/engineering/playbooks/building-for-self-hosting.mdx` for the full rule, examples (S3 OIDC, pgvector), and failure modes.
