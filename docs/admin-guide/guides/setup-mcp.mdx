## Overview

Activepieces exposes an MCP server that lets external AI agents trigger your flows and manage your tables. The MCP server uses **OAuth authentication** — you only need the **Server URL** from your **user menu → MCP** to connect.

## Connection Details

| Field | Value |
|---|---|
| **Server URL** | `https://<YOUR_DOMAIN>/api/v1/mcp` |

<Tip>
The server uses OAuth authentication. Your MCP client will be redirected to Activepieces to authorize access.
</Tip>

## Client Setup

### Cursor / JSON-based MCP Clients

If your MCP client supports `url` (e.g. Cursor), use this configuration:

```json
{
  "mcpServers": {
    "activepieces": {
      "url": "https://<YOUR_DOMAIN>/api/v1/mcp"
    }
  }
}
```

<Note>
The Server URL must end with `/mcp`.
</Note>

### Claude Desktop (mcp-remote)

Claude Desktop does not natively support HTTP transport, so use the `mcp-remote` bridge. Add this to your `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "activepieces": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote",
        "https://<YOUR_DOMAIN>/api/v1/mcp"
      ]
    }
  }
}
```

The OAuth flow will open in your browser when the client first connects.
