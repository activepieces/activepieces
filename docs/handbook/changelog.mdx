<Update label="March 2026" description="Staging Environment & CI/CD Pipeline">

### New staging environment & unified CD pipeline

We replaced the old deployment workflows with a single `continuous-delivery.yml` that introduces a staging gate before production.

**What changed:**

- Every merge to `main` now deploys to **staging** (`stg.activepieces.com`) instead of directly to production
- **Staging freeze at 5 PM UTC** — deploys pause so the content team has a stable environment overnight
- **Daily promotion at 9 AM UTC** — the frozen staging version is promoted to production (requires reviewer approval via GitHub Environment)
- **Weekly release on Sundays** — official self-hosted Docker images, git tag, and GitHub release notes are published automatically
- **Catch-up deploy** — after each promotion, staging is updated to the latest `main`

**New workflows:**

| Workflow | Purpose |
|---|---|
| `continuous-delivery.yml` | Staging deploy, daily promotion, weekly release |
| `emergency-cloud-deploy.yml` | Break-glass direct-to-prod (manual only) |
| `release-self-hosted.yml` | Build self-hosted release from any branch with a custom tag |

**Deleted workflows:**

| Workflow | Replaced by |
|---|---|
| `automate-deploy.yml` | `continuous-delivery.yml` + `emergency-cloud-deploy.yml` |
| `stg-automate-deploy.yml` | Staging path in `continuous-delivery.yml` |
| `release.yml` | Weekly release job in `continuous-delivery.yml` |
| `release-changelog.yml` | Release-drafter called inline in both release workflows |
| `pre-release-changelog.yml` | No longer needed |

**Docs added:**

- [Releases & Deployment](/handbook/engineering/playbooks/releases) — updated playbook with all workflow triggers and manual dispatch options
