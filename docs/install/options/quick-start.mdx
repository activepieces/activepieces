<Warning>
This setup is only meant for personal use or testing. It runs on [PGLite](https://pglite.dev/) (embedded PostgreSQL) and an in-memory Redis queue, which supports only a single instance on a single machine. For production or multi-instance setups, you must use Docker Compose with PostgreSQL and Redis. You will not be able to upgrade from Community Edition to Enterprise out of the box, as Enterprise does not support PGLite.
</Warning>

## Prerequisites

- [Node.js](https://nodejs.org/) v20 or later
- [Git](https://git-scm.com/book/en/v2/Getting-Started-Installing-Git)

## Install

### Step 1: Clone the Repository

```bash
git clone https://github.com/activepieces/activepieces.git
cd activepieces
```

### Step 2: Install Dependencies

```bash
npm install
```

### Step 3: Build the Project

```bash
npx turbo run build
```

### Step 4: Start Activepieces

```bash
node dist/packages/cli/src/index.js start
```

Activepieces will start and be available at [http://localhost:4321](http://localhost:4321).

### Options

| Flag | Description | Default |
|------|-------------|---------|
| `-p, --port <number>` | Port to run on | `4321` |

Example:

```bash
node dist/packages/cli/src/index.js start --port 8080
```

### Configure Webhook URL (Important for Triggers, Optional If you have public IP)

**Note:** By default, Activepieces will try to use your public IP for webhooks. If you are self-hosting on a personal machine, you must configure the webhook URL so that it is accessible from the internet.

**Optional:** The easiest way to expose your webhook URL on localhost is by using a service like ngrok. However, it is not suitable for production use.

1. Install ngrok
2. Run the following command:
```bash
ngrok http 4321
```
3. Set the `AP_FRONTEND_URL` environment variable in `~/.activepieces/.env` to the ngrok URL and restart.

### Configuration

On first run, Activepieces auto-generates secrets and stores them in `~/.activepieces/.env`. You can edit this file to override any environment variable (see [Environment Variables](/install/configuration/environment-variables)).

## Upgrade

To upgrade to the latest version:

```bash
cd activepieces
git pull
npm install
npx turbo run build
node dist/packages/cli/src/index.js start
```
