## Overview

The Code piece is where you can write your custom code. It maximizes flexibility as you can write code to achieve things you can't with the predefined pieces.

This piece currently supports writing code in **Node.js**, you can also bring in your favorite **npm packages**.

Code is executed in an isolated environment and doesn't have access to machine resources outside this environment.

## Use cases

You can do almost anything you do with regular code. For example:

- Use a package to process a CSV file and parse it as a JSON array.
- Reformat output from previous steps to a different format expected by the following pieces.
- Filter outputs from previous steps by a custom criteria.

## Instructions

As you see in the screenshot of the step settings above, you can configure the Code piece as follows:

### Parameters

To use outputs from previous steps in your code, you have to pass them through the Parameters input.

Parameters are key/value pairs. The **key** you define here will become available in your code as a property of the `params` argument in the main function.

For example, if you pass `mode` as a **key** and `full` as a **value**, you can access this parameter by `params.mode` and its value will be `full`.

The parameters become powerful when you pass outputs from previous steps. The syntax to use outputs from previous steps is generally `${step_name.variable_name}`.

For example, if you'd like to pass the dynamic output `current_mode` from the `trigger` step to a `params.mode`, you will keep the parameter **key** as `mode` and change its **value** to `${trigger.current_mode}`. This will still be accessible in your code through `params.mode`.
